# Mobile Surface

`apps/mobile/` is reserved for a future mobile experience if a dedicated mobile product becomes strategically necessary.

## Likely focus areas

If implemented, the mobile surface should prioritize:

- lightweight property discovery,
- journey status and notifications,
- document capture,
- secure approvals,
- advisor or broker collaboration touchpoints.

## Guardrail

Do not expand this directory until there is a clear product case for React Native or Flutter delivery.
